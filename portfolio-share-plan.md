# 📁 DevShelf — 포트폴리오 공유 플랫폼 기획안

> 프론트엔드 부트캠프 팀 프로젝트 | 1.0v

---

## 1. 프로젝트 개요

### 서비스 한 줄 소개
> "개발자의 포트폴리오를 한 곳에서 발견하고, 공유하고, 기록하다"

### 서비스명
**DevShelf** *(개발자들의 서재)*

- 라이트 모드: 신뢰감 있는 화이트 톤의 포트폴리오 허브
- 다크 모드: 밤늦게 코딩하는 개발자의 아늑한 서재 감성

### 핵심 가치
- **발견** — 다른 개발자의 포트폴리오를 쉽게 탐색
- **공유** — 나의 포트폴리오 링크를 등록하고 노출
- **연결** — 같은 기술 스택을 가진 개발자와 접점 형성

---

## 2. 타겟 유저

| 구분 | 설명 |
|------|------|
| 1차 타겟 | 부트캠프 수료생, 주니어 개발자 (취준생) |
| 2차 타겟 | 포트폴리오를 찾는 채용 담당자, 팀원 모집 중인 시니어 개발자 |
| 핵심 니즈 | 내 포트폴리오를 알리고 싶다 / 참고할 포트폴리오를 찾고 싶다 |

---

## 3. 핵심 기능 정의

### MVP 기능 (부트캠프 범위)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 포트폴리오 목록 조회 | 등록된 포트폴리오 카드 형태로 나열 | 🔴 필수 |
| 포트폴리오 바로가기 | 카드 클릭 → 개인 포트폴리오 사이트 새 탭 오픈 | 🔴 필수 |
| 포트폴리오 등록 | 이름, 한줄소개, 기술스택, URL, 썸네일 입력 | 🔴 필수 |
| 로그인 / 회원가입 | 이메일 + 비밀번호 기반 인증 | 🔴 필수 |
| 상세 페이지 | 포트폴리오 상세 정보 확인 | 🟡 권장 |
| 다크모드 토글 | 라이트 ↔ 다크 전환 | 🟡 권장 |
| 검색 / 필터 | 기술스택, 이름으로 검색 | 🟢 선택 |

### 📄 [PAGE 1] 메인 페이지 (Home)

**URL:** `/`  
**목적:** 서비스 소개 + 포트폴리오 목록 탐색

#### 레이아웃 구성

```
┌─────────────────────────────────────┐
│  [로고] DevShelf    [로그인] [회원가입] │  ← Header / 네비게이션
├─────────────────────────────────────┤
│                                     │
│   "개발자의 포트폴리오를 한 곳에서"    │  ← Hero Section
│   [포트폴리오 둘러보기] [내 포트폴리오 등록] │
│                                     │
├─────────────────────────────────────┤
│  [검색창] [기술스택 필터 태그들]       │  ← 필터 영역
├─────────────────────────────────────┤
│  [카드] [카드] [카드]                 │
│  [카드] [카드] [카드]                 │  ← 포트폴리오 카드 그리드
│  [카드] [카드] [카드]                 │
├─────────────────────────────────────┤
│  Footer                             │
└─────────────────────────────────────┘
```

#### 포트폴리오 카드 구성 요소
- 썸네일 이미지 (없을 경우 기본 플레이스홀더)
- 이름 + 직군 (예: 프론트엔드 개발자)
- 한 줄 소개
- 기술스택 태그 (React, TypeScript 등 최대 4개)
- "포트폴리오 보기" 버튼 → 외부 사이트 새 탭 오픈
- "상세 보기" 텍스트 링크 → 상세 페이지 이동

#### 디자인 방향

**라이트 모드**
- 배경: `#FFFFFF` / `#F8F9FA`
- 카드: 흰 배경, 미세한 그림자, 라운드 코너
- 버튼: 딥 네이비 (`#1A2B4A`) 계열 포인트
- 폰트: 세련되고 가독성 높은 산세리프
- 느낌: 깔끔한 SaaS 랜딩페이지, 신뢰감

**다크 모드**
- 배경: `#0D1117` (GitHub 다크 느낌)
- 카드: `#161B22`, 미세한 보더 (`#30363D`)
- 포인트: 앰버/골드 (`#E6B800`) — 책상 스탠드 조명 느낌
- 텍스트: `#E6EDF3`
- 느낌: 밤에 혼자 공부하는 개발자의 서재, 아늑함

## 7. 다크 모드 컨셉 상세 — "개발자의 서재"

### 비주얼 컨셉

밤늦게 조용히 공부하는 개발자의 책상. 스탠드 조명 아래 펼쳐진 기술 서적들. 여기서 포트폴리오는 하나의 **책**이고, DevShelf는 그 **책장**이다.

### 다크 모드 색상 팔레트

| 역할 | 색상 | 코드 |
|------|------|------|
| 배경 (가장 어두운) | 딥 차콜 | `#0D1117` |
| 카드/패널 배경 | 다크 그레이 | `#161B22` |
| 보더 | 미디움 그레이 | `#30363D` |
| 기본 텍스트 | 소프트 화이트 | `#E6EDF3` |
| 보조 텍스트 | 뮤트 그레이 | `#8B949E` |
| 포인트 컬러 | 앰버 골드 | `#E6B800` |
| 성공/확인 | 그린 | `#3FB950` |
| 링크 | 블루 | `#58A6FF` |

### 다크 모드 세부 인터랙션
- 카드 hover 시 상단 좌측에 미세한 앰버 보더라인 등장
- 버튼 hover 시 앰버 글로우 효과
- 배경에 미세한 노이즈 텍스처 (종이질감 연출)
- 타이포그래피에 monospace 폰트 포인트 사용 (코드 감성)

## 8. 기술 스택 (프론트엔드)

| 구분 | 선택지 | 비고 |
|------|--------|------|
| 프레임워크 | React (Vite) | CRA 대신 Vite 권장 |
| 스타일링 | Tailwind CSS | 다크모드 `dark:` prefix 활용 |
| 상태관리 | Zustand or Context API | 규모에 맞게 선택 |
| 라우팅 | React Router v6 | |
| 폼 관리 | React Hook Form | 유효성 검사 편의 |
| 아이콘 | Lucide React | |
| 모의 데이터 | JSON / LocalStorage | 백엔드 없이 구현 가능 |
